## 1. Backend Implementation (Beyond Stubs)
- **Local Source of Truth:** Maintain the offline-first approach. LifeOS is the master calendar.
- **Voice Parsing:** Integrate local STT (Speech-to-Text) using a lightweight model like `whisper.cpp` to parse natural language voice commands ("Remind me to call John tomorrow at 5") into structured Drift entries without cloud APIs.
- **Notification Daemon:** Write a background service in Go or Windows Task Scheduler to trigger OS-level push notifications for upcoming events.

## 2. Data Interconnectivity & Relationships
- **Link to Point Star System:** Checking off daily habits yields `+5 points`. Completing tasks ahead of the deadline could offer bonus multiplier points.
- **Link to Obsidian Zen Editor:** Daily tasks should automatically appear in the Daily Note markdown file generated by Obsidian.

## 3. Open Design Questions
1. Do we want unidirectional export to Google Calendar/CalDAV (so others can see your busy times), or strictly keep it air-gapped within LifeOS?
2. Should habits have "streak freezes" purchasable with Star Points?
